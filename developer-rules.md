# DEVELOPER RULES

---

# CODE QUALITY

- Clean code
- Reusable components
- Maintainable structure
- SEO friendly
- Production ready

---

# FILE STRUCTURE

/assets
    /images
    /icons

/css
    style.css
    responsive.css
    animations.css

/js
    main.js

/pages
    home
    services
    about
    blog
    contact

kebab-case

hero-section
service-card
portfolio-card
blog-card

.hero-section
.hero-title
.service-card

# HTML RULES

Use semantic HTML:
header
nav
main
section
article
footer

Example:
<header>
<nav>
<section>
<footer>

# CSS RULES

Use modular CSS

Structure:
1 Reset
2 Variables
3 Global
4 Components
5 Sections
6 Responsive

Example:
:root {
--primary: #0D7C3F;
--dark: #045B2D;
}

---

# JAVASCRIPT RULES

Use JavaScript only where needed

Allowed:
- Menu toggle
- Sliders
- Scroll animations
- Interactive UI

Avoid:
Unnecessary scripts

---

# PERFORMANCE RULES

PageSpeed target:
90+

Use:
- Lazy loading
- Minification
- Caching
- CDN

Image format:
WebP

---

# SEO RULES

Must include:
- Meta Title
- Meta Description
- Alt Tags
- Heading hierarchy

Heading Structure:
H1 only once
H2 for sections
H3 for cards

---

# ACCESSIBILITY RULES

Use:
- Proper alt text
- Button labels
- Keyboard navigation

Contrast ratio:
Accessible

---

# SECURITY RULES

Sanitize inputs
Validate forms
Secure contact form
Prevent XSS

# ANIMATION RULES

Keep animations smooth
Avoid excessive motion

Preferred:
transform
opacity

Avoid:
Heavy animations

Desktop

Mobile
